# Kontora24 — План v2.0 (с Bitrix24)

> Дата: 01.05.2026
> Ключевое решение: CRM = Bitrix24, Kontora24 = MES/WMS (производство + склад)

---

## Архитектура интеграции

```
BITRIX24 (CRM)                    KONTORA24 (MES/WMS)
──────────────                    ────────────────────

Менеджер создаёт сделку
  клиент, размер, тип,
  тираж, ламинация
        │
        ▼
Стадия "В производство"
        │
        ├──── webhook POST ────►  /api/bitrix/order-in
        │                              │
        │                         Автоматически:
        │                         • создаёт заказ
        │                         • считает калькулятор
        │                         • статус = "new"
        │                              │
        │                         Производство:
        │                         design → print → assembly
        │                              │
        │                         Статус = "done"
        │                              │
        ◄──── webhook POST ────── /api/bitrix/order-done
        │                         (стадия + цена + себестоимость)
        ▼
Стадия "Готово к выдаче"
Менеджер видит итоговую цену
```

---

## Что УДАЛЯЕМ из Kontora24

| Модуль | Причина |
|--------|---------|
| Клиенты (`/clients`) | Живут в Bitrix24 |
| КП (CommercialProposal) | Делается в Bitrix24 |
| Аналитика выручки | Bitrix24 считает деньги |
| Создание заказов вручную | Заказы приходят из Bitrix24 |

## Что ОСТАЁТСЯ

| Модуль | Роль |
|--------|------|
| Dashboard | Обзор производства (очереди, склад) |
| Калькулятор | Автоматический расчёт + ручной для прикидок |
| Заказы | Список + детали + смена статусов |
| Очереди | Дизайн / Печать / Сборка |
| Склад | Материалы, приход/расход, алерты |
| Тех карта | A4 PDF для печатника |
| Настройки | Пользователи, параметры калькулятора |

## Что ДОБАВЛЯЕМ

| Модуль | Что делает |
|--------|-----------|
| **Bitrix24 интеграция** | Webhook endpoint для приёма заказов |
| **Обратный webhook** | Уведомление Bitrix24 когда заказ готов |
| **Маппинг полей** | Настройка "какое поле Bitrix → какое поле Kontora" |
| **Производственная аналитика** | Время на этап, throughput, bottleneck (вместо денежной аналитики) |

## Что УДАЛИТЬ из кода

| Файл/модуль | Причина |
|-------------|---------|
| `src/features/clients/` | CRM = Bitrix24 |
| `src/features/kp/` | КП = Bitrix24 |
| Денежная аналитика в AnalyticsPage | Выручка = Bitrix24 |
| `/clients` route + nav item | Не нужен |
| `useClients`, `useClientOrders`, `createClient`, `updateClient` | Не нужны |
| `CommercialProposal` component | Не нужен |

Клиенты в таблице `orders` остаются как **snapshot** (имя + телефон из Bitrix), но без отдельного модуля управления.

## Структура Bitrix24 webhook

### Входящий (Bitrix → Kontora)

```json
POST /api/bitrix/incoming
{
  "bitrix_deal_id": 12345,
  "client_name": "Иван Иванов",
  "client_phone": "+7 999 123 45 67",
  "order_type": "sticker_cut",
  "width_mm": 50,
  "height_mm": 50,
  "qty": 100,
  "design_variants": 1,
  "need_lam": false,
  "deadline": "2026-05-10",
  "notes": "Срочный заказ",
  "bitrix_url": "https://your.bitrix24.ru/crm/deal/details/12345/"
}
```

### Исходящий (Kontora → Bitrix)

```json
POST https://your.bitrix24.ru/rest/webhook_token/crm.deal.update
{
  "id": 12345,
  "fields": {
    "STAGE_ID": "WON",
    "UF_COST_TOTAL": 383,
    "UF_PRICE_FINAL": 1224,
    "UF_KONTORA_URL": "https://kontora24.vercel.app/orders/uuid-here"
  }
}
```
