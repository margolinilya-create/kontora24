# Kontora24 — Полное описание проекта

> Версия: 01.05.2026
> Продакшн: https://kontora24.vercel.app
> Цель документа: описание системы для обсуждения с командой и сбора обратной связи

---

## Что это

Kontora24 — внутренняя MES/WMS система для управления стикерным производством.
Заменяет Excel, мессенджеры и устные договорённости на единый инструмент с чёткими очередями, учётом материалов и аналитикой.

**CRM** остаётся в Bitrix24 — менеджер работает там. Kontora24 получает заказы из Bitrix через webhook и отдаёт обратно статус "Готово" + цену.

**Пользователи:** 5-7 сотрудников. Работники используют свои телефоны в цеху.

---

## Команда и роли

| Человек | Роль в системе | Что видит | Что делает |
|---------|---------------|-----------|------------|
| Менеджер (ОП) | Работает в Bitrix24 | Не заходит в Kontora24 | Продажи, клиенты, сделки |
| Руководитель производства | `admin` | Всё | Назначает заказы, контролирует все этапы, настройки, пользователи |
| Дизайнер | `designer` | Свои задачи + очередь дизайна | Берёт заказ → рисует макет → отмечает "Готово" |
| Печатник | `printer` | Свои задачи + очередь печати + постобработка | Печатает на плоттере → режет/ламинирует → отмечает "Готово" |
| Заливщик 1, 2 | `resin_pourer` | Свои задачи + очередь заливки | Заливает смолой → ждёт 24ч сушки → отмечает "Готово" |
| Сборщик | `assembler` | Свои задачи + постобработка + сборка + упаковка | Собирает стикерпаки → упаковывает → отмечает "Готово" |

## Путь заказа

### Откуда приходят заказы

**Путь 1 — Из Bitrix24 (автоматически):**
```
Менеджер создаёт сделку в Bitrix24
    → Bitrix отправляет webhook в Kontora24
    → Система автоматически создаёт заказ + рассчитывает цену
    → Менеджер видит ссылку на заказ в Kontora24
```

**Путь 2 — Вручную (через калькулятор):**
```
Руководитель/Менеджер заходит в Калькулятор
    → Вводит: тип, размер, тираж, ламинация, клиент, дедлайн
    → Система рассчитывает цену
    → Нажимает "Создать заказ"
```

### Что происходит на каждом этапе

| Этап | Статус | Кто делает | Действия | Время | Оборудование |
|------|--------|-----------|----------|-------|-------------|
| Дизайн | `design` | Дизайнер | Рисует макет в Illustrator/Photoshop, готовит файл для плоттера | 15 мин — 2 ч | Компьютер |
| Печать | `print` | Печатник | Загружает файл в плоттер, печатает на плёнке | 10 — 30 мин | Широкоформатный принтер |
| Постобработка | `post_processing` | Печатник / Сборщик | Плоттерная резка (die cut или kiss cut), ламинация | 10 — 60 мин | Режущий плоттер, ламинатор |
| Заливка смолой | `resin_pouring` | Заливщик | Заливает эпоксидной смолой, ждёт высыхания | 15 мин + 24 ч сушка | Стол заливки |
| Сборка | `assembly` | Сборщик | Собирает стикерпаки | 10 — 30 мин | Руки |
| Упаковка | `packaging` | Сборщик | Упаковывает в пакеты/коробки | 5 — 10 мин | Руки |

## Типы продукции

| Тип | Описание | Наценка | Особенности |
|-----|----------|---------|-------------|
| Стикер вырубной (die cut) | Отдельный стикер, вырезанный по контуру | x4.0 | Полная вырубка через материал |
| Стикер на подложке (kiss cut) | Стикер на листе-подложке | x4.0 | Прорезается только верхний слой |
| Стикерпак | Набор из нескольких видов стикеров | x4.0 | Несколько дизайнов в одном заказе |
| 3D стикер | Стикер с эпоксидной смолой | x4.5 | Дополнительный этап заливки + 24ч сушки |
| 3D стикерпак | Набор 3D стикеров | x4.5 | То же, что стикерпак + заливка |
| Прямоугольный | Простая прямоугольная наклейка | x4.0 | Без контурной резки |
| Большой формат | Наклейка > A4 | x4.0 | Нестандартный размер |

**Наценки настраиваемые** — руководитель может изменить в Настройки → Наценки.

## Калькулятор — как считается цена

### Формула

```
1. Раскладка на листе:
   Изделий на лист = пол(1230мм / (ширина + 6мм))
   Листов = потолок(тираж / изделий на лист)

2. Расход материалов:
   Плёнка = Листов × (1230 × (высота + 30мм)) / 1 000 000  [м²]
   Краска = (тираж × ширина × высота) / 1 000 000  [м²]
   Ламинация = Плёнка (если нужна)
   Смола = (ширина × высота / 100) × 0.1444 × тираж  [г]  (только 3D)

3. Стоимость материалов:
   Плёнка: 180 ₽/м²  |  Краска: 120 ₽/м²  |  Ламинация: 120 ₽/м²  |  Смола: 1.2 ₽/г

4. Работа:
   Время резки = (периметр × тираж) / (скорость × 1000) / 3.6  [ч]
   Время ламинации = площадь плёнки / (скорость × ширину блока)  [ч]
   Время заливки = листов × 1200сек  [ч]  (только 3D)
   + 0.5ч на подготовку
   Стоимость работы = общее время × 500 ₽/час

5. Итоговая цена:
   Себестоимость = материалы + работа
   Цена = себестоимость × наценка × (1 - скидка за объём)
```

Все параметры (цены материалов, скорости, наценки) настраиваются в Настройки → Калькулятор.

### Скидки за объём

| Тираж | Скидка |
|-------|--------|
| 1 — 9 шт | 0% |
| 10 — 24 шт | 5% |
| 25 — 49 шт | 10% |
| 50 — 99 шт | 15% |
| 100 — 199 шт | 20% |
| 200 — 499 шт | 25% |
| 500+ шт | 30% |

## Склад и материалы

| Материал | Единица | Когда расходуется | Как списывается |
|----------|---------|-------------------|-----------------|
| Плёнка | м² | Печать | Автоматически при переходе в статус "Печать" |
| Краска | мл | Печать | Автоматически при переходе в статус "Печать" |
| Ламинация | м² | Постобработка | Автоматически (если ламинация нужна) |
| Смола | г | Заливка | Вручную (работник записывает расход) |
| Упаковочные пакеты | шт | Упаковка | Вручную |
| Коробки | шт | Упаковка | Вручную |

**Автосписание:** при переходе заказа в статус "Печать" система вызывает функцию `auto_deduct_materials`, которая рассчитывает расход плёнки, краски и ламинации по формулам калькулятора и списывает со склада.

**Резервирование:** при создании заказа материалы резервируются. При отмене — возвращаются.

**Предупреждения:** если остаток материала <= минимуму, в боковой панели и на дашборде появляется значок "Низкий остаток".

## Интеграция с Bitrix24

### Входящий webhook (Bitrix → Kontora24)

```
Bitrix24 создаёт сделку
    → POST /api/bitrix/incoming
    → Kontora24 создаёт заказ + считает цену
    → Возвращает ссылку на заказ в Kontora24
```

### Исходящий webhook (Kontora24 → Bitrix)

При каждой смене статуса:
```
Kontora24 меняет статус заказа
    → POST /api/bitrix/status-update
    → Bitrix обновляет стадию сделки

Маппинг статусов:
  new           → NEW (Новая)
  design, print → EXECUTING (В работе)
  done          → WON (Выиграна) + обновляет цену
  cancelled     → LOSE (Проиграна)
```

### Настройки

В Настройки → Bitrix24:
- URL вебхука
- Включить/выключить интеграцию
- Маппинг полей (какое поле Bitrix → какое поле Kontora24)

## Аналитика

Три вкладки:

### Финансы
- Выручка за период (график по дням)
- Выручка по типам продукции (столбчатая диаграмма)
- Средний чек
- Маржинальность
- Сравнение с предыдущим периодом (рост/падение в %)

### Производство
- Распределение по статусам (круговая диаграмма)
- Среднее время на этап (bottleneck detection)
- Конверсия воронки (% заказов new → done)
- Throughput по неделям (сколько заказов завершается)

### Ресурсы
- Расход материалов за период
- Загрузка по исполнителям (кто сколько завершил)
- Топ-10 клиентов по выручке

Все отчёты можно экспортировать в PDF.

## Карточка заказа

Каждый заказ содержит:

| Поле | Описание |
|------|----------|
| Номер | Автоматический (#1, #2, ...) |
| Тип | Стикер вырубной / на подложке / стикерпак / 3D / прямоугольный / большой формат |
| Размер | Ширина × Высота (мм) |
| Тираж | Количество штук |
| Кол-во видов | Сколько разных дизайнов |
| Ламинация | Нужна / не нужна (глянцевая / матовая) |
| Клиент | Имя, телефон, email |
| Дедлайн | Дата |
| Приоритет | Низкий / Обычный / Высокий / Срочный |
| Исполнитель | Назначенный работник |
| Заметки | Свободный текст |
| Bitrix ID | Ссылка на сделку в Bitrix24 |

### На странице заказа доступно:
- **Редактирование** всех полей
- **Комментарии** — лента сообщений в реальном времени
- **Файлы** — загрузка макетов, дизайнов (Supabase Storage)
- **Таймлайн** — визуальная шкала прохождения по этапам
- **Тех карта** — экспорт в PNG / PDF / печать + QR-код
- **Коммерческое предложение** — 3 шаблона (стандартный / подробный / краткий)
- **Расход материалов** — запись фактического расхода
- **Чеклист операций** — отметки выполненных операций
- **Учёт времени** — старт/стоп таймер
- **Пересчёт цены** — если изменились настройки калькулятора
- **Смена статуса** — переход на следующий этап
- **PDF экспорт** — полная карточка заказа

## Известные ограничения

1. **Bitrix интеграция не подключена** к реальному Bitrix24 — готова технически, нужно настроить webhook URL
2. **Дублирование заказа** — кнопка "Повторить" открывает калькулятор с параметрами, но не создаёт копию автоматически
3. **Webhook-эндпоинты не пишут логи** в таблицу integration_log (UI для просмотра есть, записи нет)
4. **Нет Telegram-уведомлений** — только звуковые в браузере
5. **Нет штрих-кодов** для инвентаризации склада
